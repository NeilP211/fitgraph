# FitGraph Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a local, runnable outfit-compatibility GNN: HGAT over a garment↔outfit graph, contrastive training, FastAPI inference with pgvector retrieval, a Redis-Streams feedback loop, and a Next.js frontend.

**Architecture:** A single Python package `fitgraph` (data → embeddings → graph → model → training → eval → retrieval → db → feedback → api), plus a Next.js `web/` app. Postgres (pgvector) and Redis run via Docker Compose. The HGAT learns compatibility-aware item embeddings from outfit co-occurrence; the API embeds an uploaded garment and does ANN search over the catalog.

**Tech Stack:** Python 3.12, PyTorch + PyTorch Geometric, open_clip, sentence-transformers, FastAPI, SQLAlchemy + psycopg, pgvector, redis-py, pytest, ruff; Next.js 15 + TypeScript + Tailwind; Docker Compose; GitHub Actions.

---

## Conventions

- Tests: `pytest` from repo root. Python lint: `ruff check src tests`.
- Each task ends with a build/test run and a `git commit`.
- Real datasets only; subset size controlled by `config.py` / CLI flags.
- Device selection: prefer MPS (Apple Silicon), fall back to CPU.
- Never commit `data/`, checkpoints, or `node_modules/` (covered by `.gitignore`).

  eval/

  retrieval/

  db/

  api/

## Phase 3: Embeddings

**Files:** `embeddings/clip_encoder.py`, `embeddings/text_encoder.py`, `embeddings/fusion.py`, `scripts/build_embeddings.py`, tests.

- [ ] `clip_encoder.py`: `ClipEncoder` wrapping open_clip ViT-B/32; `encode_images(paths, batch_size) -> np.ndarray`. MPS-aware. Test with 2 small fixture images: output shape `(2, 512)`, finite values.
- [ ] `text_encoder.py`: `TextEncoder` wrapping `sentence-transformers/all-MiniLM-L6-v2`; `encode(texts) -> np.ndarray`. Test output shape.
- [ ] `fusion.py`: `fuse(img, txt) -> np.ndarray` concat + L2-normalize. Test norms ≈ 1, shape = img_dim+txt_dim.
- [ ] `scripts/build_embeddings.py`: load items for the configured subset, encode, save `data/embeddings/items.npz` (ids + vectors). **Run for real** on the subset.
- [ ] Verify: `ruff check && pytest -q`; embeddings file exists with expected row count. Commit `feat: CLIP + text embeddings with feature fusion`.

## Phase 4: Graph

- [ ] `scripts/build_graph.py`: build from subset artifacts, save `data/graph/graph.pt`. Run for real.
- [ ] Verify: `ruff check && pytest -q`. Commit `feat: bipartite hetero graph builder`.

## Phase 5: Model + training

**Files:** `models/hgat.py`, `training/loss.py`, `training/negatives.py`, `training/trainer.py`, `scripts/train.py`, tests.

- [ ] `hgat.py`: `HGAT` using PyG `HeteroConv` + `GATConv` layers; input proj from fused-embedding dim → hidden; produces per-garment embeddings. Test: forward on fixture graph returns `(num_garments, hidden)`, gradients flow.
- [ ] `loss.py`: `info_nce(anchor, positive, negatives, temperature)`. Test: identical anchor/positive with random negatives yields low loss; shuffled yields higher.
- [ ] `negatives.py`: `mine_hard_negatives(anchor_ids, embeddings, co_occurrence, k)` — picks CLIP-near items not co-worn. Test on fixture: returned ids exclude true co-occurrences.
- [ ] `trainer.py`: `Trainer.fit()` — epochs, optimizer, val AUC each epoch, checkpoint best to `models/vN/`, write `model_versions` metadata; optional W&B. Test a 2-epoch smoke run on the fixture graph: loss decreases, checkpoint written.
- [ ] `scripts/train.py`: CLI (`--full`, `--epochs`). **Run a real subset training run**; capture final val AUC.
- [ ] Verify: `ruff check && pytest -q`; report real training metrics. Commit `feat: HGAT model and contrastive training loop`.
